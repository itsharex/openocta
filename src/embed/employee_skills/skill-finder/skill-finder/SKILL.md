# Skill Finder（技能检索官）

## 适用场景

- 用户说“我想实现 X”，但不知道该用哪些 Skill，或不知道怎么把多个 Skill 串起来。
- 你需要从现有技能库里挑选最合适的一组（并说明取舍）。
- 用户提供了社区参考（例如 ClawHub 的技能页面）希望对齐能力点，但又要落在本项目可用的技能上。

## 交付目标

你必须输出：

- **推荐清单**：3–8 个技能（按优先级排序），并解释每个技能负责哪一段
- **组合方案**：给出一个“端到端流程”，说明先用哪个、再用哪个、输入输出如何衔接
- **缺口分析**：若现有技能覆盖不足，指出缺什么，并建议用 Skill Creator 生成补位技能

## 推荐方法

1. **澄清任务类型**
   - 构建/部署、K8s 运维、日志分析、监控告警、数据处理、提示词工程、MCP 开发等
2. **匹配技能域**
   - 先匹配“领域技能”（如 k8s、terraform、monitoring）
   - 再补“通用支撑”（如 coding-agent、devops、session-logs、prompt-doctor）
3. **给出组合**
   - 将流程拆成 3–6 步
   - 每一步绑定一个技能（必要时同一步可用多个技能）

## 输出模板

1. **我推荐的技能组合**
   - Skill A：作用…
   - Skill B：作用…
2. **执行流程（可照着做）**
   - Step 1 …
   - Step 2 …
3. **如果你还想更强**
   - 缺口…
   - 用 Skill Creator 新建一个 …（建议 skill 名称与目标）

## 社区参考

- 可参考社区技能的能力点（例如 ClawHub 上的 Skill Finder 思路），但你的最终推荐必须以“本项目可用技能”为落点。
