# Sandbox Guard（沙箱守卫）

## 适用场景

- 你需要检查「沙箱配置」是否足够安全：命令白名单/黑名单、参数约束、工作目录、网络访问、文件系统访问、超时与资源限制等。
- 你需要为某个工作流/Skill 的命令执行设计 **安全钩子** 与 **命令校验**，避免危险操作（删库、窃取凭证、数据外传、任意代码执行）。
- 你需要把安全策略写成清晰可审计的规则，并给出可落地的调整建议。

## 输出要求（你必须产出）

- **风险清单**：逐条列出风险点（危害、触发条件、影响范围、严重程度）。
- **策略建议**：最小权限原则下的具体建议（允许/禁止的命令、参数范围、目录、网络、环境变量）。
- **可执行改动**：给出可直接粘贴到配置中的建议（以“配置片段”或“变更说明”的形式），避免空泛建议。

## 分析步骤

1. **收集信息**
   - 目标系统与执行环境（OS、shell、容器/本机）
   - 允许的任务类型（构建、测试、部署、数据处理等）
   - 当前沙箱配置（hooks、validators、允许的命令/路径/网络）
2. **命令面审计**
   - 找出高风险命令：`rm`、`dd`、`mkfs`、`chmod/chown`、`curl/wget`、`ssh/scp`、`docker`、`kubectl`、`openssl`、`python/node` 执行任意脚本等
   - 识别危险参数：`-rf`、`--privileged`、`--mount`、`--net=host`、`-o ProxyCommand` 等
3. **数据外传审计**
   - 网络出口限制（域名/IP 白名单）
   - 访问密钥/凭证的路径与环境变量暴露
4. **文件系统审计**
   - 允许读写目录（workspace、临时目录）与禁止目录（`~/.ssh`、`/etc`、`/var/run` 等）
5. **给出修复**
   - 优先“收紧默认值”：默认拒绝，按需放行
   - 使用明确的 allowlist + 参数约束 + 超时/资源限制

## 规则建议（通用）

- 默认禁止：
  - 任意网络访问；只允许到明确域名/IP（最好带端口与协议）
  - 访问敏感目录：`~/.ssh`、`~/.aws`、`~/.config`、`/etc`、`/var/run`、`/proc`、`/sys`
  - 特权容器/宿主逃逸相关命令与参数
- 对允许命令进行 **参数白名单**：
  - 例如允许 `git` 但禁止 `git config --global` 与任意 `!` shell escape
- 所有执行都要有：
  - 超时、最大输出、最大并发、最大文件大小（如有）

## 交付模板

你最终回答用下面结构：

1. **结论（一句话）**
2. **风险清单（表格）**
3. **建议策略（分点）**
4. **建议配置片段 / 变更说明（可复制）**
